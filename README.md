# GATE Preparation Tracker

A robust, minimal, offline-first execution suite optimized for managing massive data trajectories through your GATE curriculum without generating cognitive debt. Engineered to reflect the density and speed of premium Native Desktop apps (e.g., Linear, Apple utilities).

## Key Features

- **Decentralized Analytics Engine**: Generates real-time performance heuristics locally through mathematically sound moving-averages.
- **Offline Reliability (IndexedDB)**: Natively utilizes Dexie.js to ensure lightning-fast read/writes ensuring 100% data access regardless of latency.
- **Dense Trajectory Tracking**: Strict `1200px-1400px` multi-column desktop constraints mapping structural test tracking, PYQ vulnerabilities, and micro-session limits without endless scrolling.
- **Biased Recommendation Weighting**: Automatically ranks your failing subjects leveraging exponential recency-decay formulas directly bound to Mock Test errors.

## Local Setup Instructions

Ensure you have Node.js installed, then execute:

```bash
# 1. Clone the repository natively (or extract files)
npm install

# 2. Run the offline testing suite
npm run dev

# 3. Create absolute production deployments locally
npm run build
```

Pushing this offline vault to GitHub takes 4 precise operations. Open your terminal at the project root and execute exactly:

```bash
# Initialize git architecture natively
git init

# Add all offline structures
git add .

# Save the exact base configuration

# Bind strictly to standard branch rules
git branch -M main

# Connect to GitHub (Replace YOUR_URL with your actual repo link)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY.git

```

## Vercel CI/CD Production Protocol

Deploying to Vercel provides completely free global edge-hosting, natively parsing our specific SPA `<ErrorBoundary>` route rules defined precisely within `vercel.json`.

1. Login to [Vercel](https://vercel.com/) and click **Add New Project**.
2. Select your repository from the native GitHub connection interface.
3. Validate the `Build and Output Settings` strictly mirror these constraints:
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. Click **Deploy**. Vercel will process the native `vercel.json` SPA lock preventing internal 404 router bleed.

### Live Updating
Any future git pushing (e.g. `git push origin main`) natively signals Vercel to rebuild your `dist` map generating live updates in seconds.
